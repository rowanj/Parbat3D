ROI class

The ROI class stores all the information to do with a Region of Interest. This includes its colour, name and a vector of entities that is consists of.

Private Variables

bool active

The active boolean is used to determine if a Region of Interest is visible on screen. If it is set to true then the Region of Interest will be visible, but if it is false the Region of Interest will not be shown. It is also used to determine if the Region of Interest will be shown in the feature space. Regions of Interest that have their active boolean set to false will not be coloured in the feature space.

int colour_red, colour_green, colour_blue

These three integers are used to represent the red, green and blue colour intensities that are combined to represent the colour of the Region of Interest. Each value must be in the range of 0 to 255.

string name

The name of the Region of Interest is stored in this string. The name that is used has no restrictions on it from this classes point of view, but there are some restrictions imposed by other classes, as is mentioned in the Region of Interest section of the user manual.

vector entities

This vector stores pointers to all of the entities that make up this Region of Interest.

Public Methods

ROI (void)

Creates a Region of Interest that is visible, has no name and is set to the default colour (grey).

~ROI (void)

Destructor clears the vector of all the entities that make up this Region of Interest.

void set_name (string new_name)

Sets the name of the Region of Interest. The name that is passed in as a paramter will not be checked if it is valid. This means that the responsibility of ensuring that the name is legitimate is placed on the class that is implementing this, such as ROISet.

string get_name (void)

Returns the name of this Region of Interest. The returned value will be whatever is stored in the name string.

void set_active (bool isActive)

Sets whether this Region of Interest is currently active. The boolean value passed in will be set to the active variable.

bool get_active (void)

Returns whether the ROI is currently active. This method just returns the active boolean.

void set_colour (int red, int green, int blue)

Sets the colour of this Region of Interest based on red, green and blue values that are passed in. Each of the parameters will be set to colour_red, colour_green and colour_blue as appropriate. The values are not checked to see if they are in range so the caller should ensure that legitimate values are used.

void set_color (int red, int green, int blue)

Does the exactly same a set_colour() but is included to allow for American spelling.

void get_colour (int* red, int* green, int* blue)

Allows the user to get the 3 colours that make up this Region of Interest by sending in pointers they want the values to be set to. Each of the values stored in colour_red, colour_green and colour_blue will be set to the pointers.

void get_color (int* red, int* green, int* blue)

Does the exactly same a get_colour() but is included to allow for American spelling.

void add_entity (ROIEntity* entity)

Adds an entity to this Region of Interest. There are no checks performed on the entity that is passed in as a parameter so the caller must ensure that it is valid. The pointer to the entity will be added to the end of the vector.

vector ROI::get_entities (void)

Returns the vector consisting of all the entities that make up this Region of Interest.
