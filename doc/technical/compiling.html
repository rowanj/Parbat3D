Compiling Parbat3D

Parbat 3D is developed using Bloodshed Dev-C++

Obtaining the Source

The development source tree for parbat is publically available from own SubVersion repository at:
https://svn.sourceforge.net/svnroot/parbat/trunk

In order to download the latest version of the source code, you will need to obtain an SVN client, such as TortiseSVN and follow its instructions to download from the above URL.

Setting up your Development Environment

To compile from source, you will need to have the GDAL libraries and headers installed. You can obtain these as a ZIP archive from the developer package on the SourceForge.net file release system.

You will need to un-pack the archived directory into the top level of the Parbat3D source tree.

In order to compile your software you will also need to a copy of the Bloodshed Dev-C++ IDE. This document will describe how to compile using version 4.9.9.2.

Setting up your Runtime Environment

In order to actually run the compiled executable, you will need to unzip the DLL runtime archive from our SourceForge.net file release system into the directory you have the copiled executable in (by default, the top-level build directory).

We have taken great care to package all the required libraries, although we may have missed one or two, if you have any further library problems at run-time, please post in our support forums.

Compiling The Source Code

Once you have setup your development and runtime environments and downloaded the source code, open the project file parbat3d.dev in Dev-C++. The parbat3d.dev file should be located in the top level of the Parbat3D source tree.

Once the Parbat3D project file has been opened, click on 'Compile & Run' from the 'Execute' menu within Dev-C++.
