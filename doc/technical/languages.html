Parbat3D makes use of several programming languages and APIs.

C++ and the C++ Standard Template Library

Parbat3D is written in C++. It uses standard C++ syntax and calling methods, expect in the following areas:

  • Some GDAL calls have been written as C function calls rather than C++, even though GDAL supports this. This is due to linking problems with MinGW/Dev-C++. There is no loss of functionality due to this, in so far as we use GDAL.
  • The Feature Space class uses a deprecated header, hashmap. This header is a part of SGI's additions to the STL, and has been merged into other parts of the STL. This should be able to be changed with little effort, but is currently not an issue.

C++ and the STL are documented via the various standards bodies (ANSI, ISO, etc). This author recommends O'Reilly technical volumes.

OpenGL

Standard OpenGL language calls have been used as far as possible. Window supports an older implementation of the OpenGL standard, and some features have been accessed via driver-specific extensions. Some graphics hardware may not work as expected with some Parbat3D functionality, but most ATi/nVidia chipsets should work fine.

Intel Shared Graphics has a known performance issue with smooth points; these should not be used.

OpenGL is documented on the OpenGL website.

Win32

Parbat3D uses the Win32 API. This is mostly contained in the View package and the Settings package, which directly interface with the OS for windowing and disk IO.

Documentation on the Win32 API can be found at Microsoft's Developer website.

GDAL

Parbat3D leverages the power of the Geospatial Data Abstraction Library (GDAL). It is used for all image file disk IO.

GDAL is documented here.
