View Package

The view package contains the main source files for handling the graphical user interface and communicating with the Microsoft Windows API. It consists of a class for each main window, a class for each tab in the tool window, a class to handle snapping the main windows together and some other general windowing classes.

Windows

Each of the main windows have their own class that is subclassed from a general Window class.

The Window class handles generating each main window through the Windows API and associates them with their own c++ object. This allows static event handlers to directly access the main window objects and their variables.

There is a message handler in the Window class which handles all events for all of the main windows by default. The main functions of this handler is handling the global shortcut keys and snapping and moving snapped windows (through the sticky window manager). The class has been written in a way that allows multiple message handlers to process messages in a chain. Each of the main windows use this feature to process some of their own events, while allowing the Window class to process everything else. The majority of the messages that the main windows process are passed on to Window class. The messages that are not processed by the Window class are then passed onto the Window API to perform the default operation.
