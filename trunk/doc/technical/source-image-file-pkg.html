BandInfo

BandInfo store all the information that GDAL returns about a given raster band. It requires a GDALRasterBand as an argument to its constructor. It has functions to return all different sorts of info, including the band itself.

Theoretically, bands in GDAL may have special metadata attached to them, but so far I have not been successful in reading anything other than the basic information.

BandInfo stores and returns the following: minimum data value, maximum data value, raster block x and y dimensions, the number of overviews, the band number and, finally, the band object itself.

CoordinateInfo

GDAL has the capability to read and interpret geospatial coordinate transform data from files. Parbat3D will read this data as well, but it currently does not use it.

This class simply stores an array of 6 doubles, but does not return them.

It is expected that this class will be extended in the future.

ImageFile

ImageFile is the IO workhorse of Parbat3D, but is actually quite simple. Most of the effort in making this work was through the trial and error of getting GDAL to link.

ImageFile takes a filename as an argument and contains (upon loading the file metadata) an object of class ImageProperties (see below), an STL vector of object of type BandInfo (see above), an object of type CoordinateInfo (see above), a filename (not strictly necessary), a GDALDataset pointer and an info string.

Apart from methods returning these various data objects, ImageFile has one major method: getRasterData. This method uses GDAL to read the actual pixel data from the file. It is currently flawed in that it only reads 8 bit (single byte) image depths. This can be changed easily, but would require major retooling of other sections of Parbat3D.

It is expected that this will be done at some point in the future.

ImageProperties

ImageProperties stores all relevant properties of a given image file.

The properties are the width and height of the image, the number of bands, the driver name, the full driver name (a somewat longer and more accurate string), and the file name.

ImageProperties takes a file name (as a pointer to an array of characters) and a GDALDataset pointer as its arguments.
